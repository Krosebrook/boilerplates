# Boilerplates CLI Roadmap

> A comprehensive development roadmap from current state to post-MVP maturity.

## Current State (v0.0.7)

### Completed Features
- [x] Core CLI framework with Typer
- [x] Template system with Jinja2 rendering
- [x] Multi-library support (git + static)
- [x] Schema versioning (1.0, 1.1)
- [x] Docker Compose module with 33 templates
- [x] Rich interactive prompts
- [x] Semantic validation system
- [x] Custom exception hierarchy
- [x] DisplayManager with Nerd Font icons
- [x] ConfigManager for user preferences
- [x] Repository sync for git libraries
- [x] Variable dependencies and sections
- [x] Autogenerated values (passwords, UUIDs)
- [x] Python 3.9-3.14 compatibility

### Known Issues
- [ ] PyPI publishing disabled (GitHub Releases only)
- [x] NixOS flake version mismatch (fixed)
- [ ] Some templates marked as draft

## Contributing to the Roadmap

We welcome community input on roadmap priorities:

1. **Feature Requests**: Open an issue with `[Feature]` prefix
2. **Template Requests**: Open an issue with `[Template]` prefix
3. **Bug Reports**: Open an issue with `[Bug]` prefix
4. **Discussions**: Use GitHub Discussions for ideas

Priority is determined by:
- Community demand (issue reactions)
- Maintenance burden
- Strategic alignment
- Contributor availability
